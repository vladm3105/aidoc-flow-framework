# Review Document Standards for Autopilot Skills

## Purpose

This document defines the mandatory standards for review reports generated by all autopilot skills. Review reports are formal project documents and MUST be stored alongside the reviewed documents, not in temporary folders.

**Applies to**: All `doc-*-autopilot` skills

---

## 1. Storage Location (MANDATORY)

Review reports MUST be stored in the same location as the reviewed document:

| Document Structure | Review Report Location |
|-------------------|------------------------|
| Monolithic document | Same folder as the document |
| Sectioned document | Inside the section folder |

**Examples**:

```
# Monolithic BRD
docs/01_BRD/
├── BRD-01_platform.md
└── BRD-01.R_review_report.md         # ← Review stored here

# Sectioned BRD
docs/01_BRD/BRD-03_f3_observability/
├── BRD-03.0_index.md
├── BRD-03.1_core.md
├── BRD-03.2_requirements.md
├── BRD-03.3_quality_ops.md
└── BRD-03.R_review_report.md         # ← Review stored here

# PRD
docs/02_PRD/
├── PRD-01_feature.md
└── PRD-01.R_review_report.md         # ← Review stored here
```

**NEVER store review reports in**:
- `tmp/` folder
- Root project directory
- A separate `reviews/` folder

---

## 2. File Naming Convention (MANDATORY)

**Pattern**: `{ARTIFACT}-NN.R_review_report.md`

| Component | Description | Example |
|-----------|-------------|---------|
| `{ARTIFACT}` | Artifact type code | BRD, PRD, ADR, SYS, REQ, etc. |
| `-NN` | Document number | -01, -03, -12 |
| `.R` | Review document suffix | Always `.R` |
| `_review_report` | Document type descriptor | Always `_review_report` |
| `.md` | File extension | Always `.md` |

**Examples by Artifact Type**:

| Artifact | Review Report File Name |
|----------|-------------------------|
| BRD-01 | `BRD-01.R_review_report.md` |
| BRD-03 | `BRD-03.R_review_report.md` |
| PRD-01 | `PRD-01.R_review_report.md` |
| ADR-05 | `ADR-05.R_review_report.md` |
| SYS-02 | `SYS-02.R_review_report.md` |
| REQ-03-001 | `REQ-03-001.R_review_report.md` |
| SPEC-01 | `SPEC-01.R_review_report.md` |
| TSPEC-01 | `TSPEC-01.R_review_report.md` |
| TASKS-01 | `TASKS-01.R_review_report.md` |

## 3. YAML Frontmatter (MANDATORY)

All review reports MUST include YAML frontmatter with the following fields:

```yaml
---
title: "{ARTIFACT}-NN.R: {Module Name} - Review Report"
tags:
  - {artifact-type-lowercase}     # e.g., brd, prd, adr
  - {module-type}                 # e.g., foundation-module, domain-module
  - layer-{N}-artifact            # Layer number
  - review-report                 # Always include
  - quality-assurance             # Always include
custom_fields:
  document_type: review-report
  artifact_type: {ARTIFACT}-REVIEW    # e.g., BRD-REVIEW, PRD-REVIEW
  layer: {N}                          # Layer number (1-11)
  parent_doc: {ARTIFACT}-NN           # e.g., BRD-03, PRD-01
  reviewed_document: {full_slug}       # e.g., BRD-03_f3_observability
  module_id: {module_id}              # e.g., F3, D1
  module_name: {module_name}          # e.g., Observability, Agent Orchestration
  review_date: "YYYY-MM-DD"           # ISO date format
  review_tool: doc-{type}-autopilot   # e.g., doc-brd-autopilot
  review_version: "{version}"         # Tool version
  review_mode: read-only              # or fix-mode
  {score_field}_claimed: {claimed}    # e.g., prd_ready_score_claimed: 92
  {score_field}_validated: {validated} # e.g., prd_ready_score_validated: 85
  validation_status: PASS|FAIL
  errors_count: {n}
  warnings_count: {n}
  info_count: {n}
  auto_fixable_count: {n}
---
```

**Score Field Names by Artifact**:

| Artifact | Score Field Prefix |
|----------|-------------------|
| BRD | `prd_ready_score` |
| PRD | `ears_ready_score` |
| EARS | `bdd_ready_score` |
| BDD | `adr_ready_score` |
| ADR | `sys_ready_score` |
| SYS | `req_ready_score` |
| REQ | `spec_ready_score` |
| SPEC | `tasks_ready_score` |
| TSPEC | `impl_ready_score` |
| TASKS | `code_ready_score` |

## 4. Section Structure (MANDATORY)

All review reports MUST follow this section structure:

| # | Section | Content |
|---|---------|---------|
| 0 | Document Control | Report metadata, review date, tool version |
| 1 | Executive Summary | Overall score, status, issue counts |
| 2 | Document Overview | Reviewed document details, files reviewed |
| 3 | Score Breakdown | Category-by-category scores with max values |
| 4+ | Validation Details | Artifact-specific validation sections |
| N-3 | Issues Summary | Errors, warnings, info - categorized lists |
| N-2 | Recommendations | Priority-ordered fix recommendations |
| N-1 | Fix Mode Command | Commands to run fix mode |
| N | Traceability | Parent document reference |

---

## 5. Parent Document Reference (MANDATORY)

Every review report MUST include:

1. **Navigation Header**:
   ```markdown
   > **Navigation**: [Parent: {ARTIFACT}-NN Index]({relative_path_to_index})
   > **Review Type**: Validation Report (Read-Only)
   > **Generated**: YYYY-MM-DD
   ```

2. **Traceability Section**:
   ```markdown
   ## N. Traceability

   ### Parent Document

   | Reference | Document |
   |-----------|----------|
   | @parent-{type} | {ARTIFACT}-NN |
   | Parent Index | [{ARTIFACT}-NN.0_index.md](relative_path) |
   | Parent Location | `docs/NN_{TYPE}/{ARTIFACT}-NN_{slug}/` |

   ### Review Context

   | Item | Value |
   |------|-------|
   | Review Trigger | User command or CI/CD |
   | Review Scope | Full document validation |
   | Sections Reviewed | List of section files |
   ```

## 6. Content Requirements

### 6.1 Issues Must Be Categorized

| Category | Severity | Description |
|----------|----------|-------------|
| ERROR | Blocking | Must fix before downstream artifact generation |
| WARNING | Non-blocking | Should fix, but doesn't block |
| INFO | Informational | Suggestions for improvement |

### 6.2 Auto-Fixable Issues Must Be Identified

For each issue, indicate:
- Whether it can be auto-fixed
- The fix action that will be applied
- Location (file:line if applicable)

### 6.3 Recommendations Must Be Prioritized

| Priority | Description |
|----------|-------------|
| Priority 1 | Must fix before downstream generation |
| Priority 2 | Should fix before downstream generation |
| Priority 3 | Future improvements |

## 7. Integration with Autopilot Skills

Each autopilot skill MUST:

1. **Generate review reports** that comply with these standards
2. **Store review reports** alongside the reviewed document
3. **Include proper metadata** in YAML frontmatter
4. **Reference the parent document** in Traceability section
5. **Not store review reports** in temporary folders
